Hello friends!! I've got some exciting announcements in the form of updates, services, and products.

## What do I mean by Service?
A service is a platform a user can signup for, pay a set fee, then receive something in return. This can be as expansive as an entire website (see [GroveEngine](https://github.com/AutumnsGrove/GroveEngine)), or as simple as 5 products searched online.

## New stuff
Okay!! enough boring banter. Lets talk about the real deal. What have I been working on?

- The first is a service you are already familiar with, the [GroveEngine](https://github.com/AutumnsGrove/GroveEngine). This is a service where users can signup for a personal blog and share ideas/thoughts/whatever with their friends.
- The second is [GroveScout](https://github.com/AutumnsGrove/GroveScout). This service will search across multiple retailers like Amazon, Walmart, etc. to reduce shopping overwhelm for users and provide simple, digestible, results.
- Next up I have the [GroveDomainTool](https://github.com/AutumnsGrove/GroveDomainTool). This is yet another agentic searching tool but specifically for domain names and availability. I use this internally when helping clients migrate from <username>.grove.place -> personal domains. 

The next two services are currently under construction and will not be publicly accessibly for a bit while I develop them further.

- [GroveMusic](https://github.com/AutumnsGrove/GroveMusic) is a tool users can signup for to get curated and personalized playlists based on seed tracks, artists, or albums. Creates a ~15 song list with a weighted average of obvious picks, deep cuts, and hidden gems.
- [GroveSearch](https://github.com/AutumnsGrove/GroveSearch) is another agentic tool for searching the web and getting fully in depth answers while maintaining factual completeness.

Development wise (and this will be a bit technical, skip ahead if you're not interested in Coding), I've become particularly enamored by the recently released [Deepseek v3.2](https://arxiv.org/abs/2512.02556) model due to its price and excellent performance. I've also started using [Kilo Code](https://kilo.ai/), a similar agent to something like Cursor but its open source and very easy to use, and doesn't require an entire separate IDE, instead installing within an extension. I'm experimenting with using Kilo more full time, as my usage of [Claude Code](https://www.claude.com/product/claude-code) continues to drain my wallet (an example task, ~20 tool calls with ~60k tokens of context will cost deepseek ~0.03$. Claude code is ~2$ for the same task)

Kilo doesn't _fully_ compare to Claude Code, Anthropic is really cooking right now, but its closing the gap. I'm also experimenting with enabling other models to be used within claude code itself (like deepseek) via the [Claude Code Router](https://github.com/musistudio/claude-code-router) project.
